Normalization of data

RStudio
Inspection

Levels of normalization of data structure…

Author

Søren O’Neill

Published

November 26, 2023

To manage your data effectively and safely, it is a good idea to know a little bit about database normalization: “Normalization is a process that database designers use to eliminate data redundancy, improve data integrity, and enhance the overall efficiency of the database.” 1

The normalization of data has nothing to do with the normal (Gaussian) distribution.

Data normalization it is not a case of either/or – data can be normalized to different levels, depending on your needs.

1 First Normal Form – 1NF

The First Normal Form requires that:

  • Each column represents one variable
  • Each variable contains atomic values – i.e. the smallest (indivisible) unit of information
  • Variables to not contain arrays of atomic values
  • The data set does not contain repeating groups of similar variables

2 Second Normal Form – 2NF

The Second Normal Form requires that data conforms to the 1NF requirements and additionally:

  • Data contains a single-column primary key.
  • That all non-key variables ‘depend’ on the entire primary key.

Example 1: The data does not contains a single-column primary key. Each observation is unique and identifiable by the (compound) primary key subject_id + test_id.

Example 2: The data does contains a single-column primary key (key), but all non-key variables do not ‘depend’ on the entire primary key. Specifically, the variable type is contingent exclusively on test_id. In this example, there is a very simple \(1:1\) relationship between test_id and type (test ‘a’ is always ‘Blood’, and vice versa), but that need not be the case, and the test type could include other information than simply Blood versus Saliva.

Example 3: This data structure conforms to Second Normal Form (2NF) data structure. The data has been split into two separate tables, with no redundancy of information. The variable test_id in the first, larger data set, corresponds to test_id in the second, smaller data set. This structure makes data much easier to survey, when the amount of data grows large.
