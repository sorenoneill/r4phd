Difference between == and %in%

tldr - Use %in% instead of == when you want to filter for multiple values.

Read on if you want to understand why. (You don‚Äôt have to)

The code filter(Ethnicity == c("Mexican", "Filipino")) is likely not doing what you expect. The ‚Äò==‚Äô operator does an element-wise comparison, which means it compares the first element of ‚ÄòEthnicity‚Äô to the first element of the vector (‚ÄúMexican‚Äù), the second element of ‚ÄòEthnicity‚Äô to the second element of the vector (‚ÄúFilipino‚Äù). The short vector is then recycled so now the third element of ‚ÄòEthnicity‚Äô is compared to the first element of the vector (‚ÄúMexican‚Äù), the fourth element of ‚ÄòEthnicity‚Äô to the second element of the vector (‚ÄúFilipino‚Äù), and so on.

Inspect the differences in how may rows these lines of code produce

soldiers %>% 
  filter(Ethnicity %in% c("Mexican", "Filipino"))

soldiers %>% 
  filter(Ethnicity == c("Mexican", "Filipino"))

Run this code chunk line by line. Inspect the differences.

# Create a data frame
df <- data.frame(

  stringsAsFactors = FALSE
)

# Investigate the data frame
df

# Filter using ==

2.6 count()

For simple counting count() is faster than summarise(n = n()) or mutate(n = n())



What is this code equivalent to?

diamonds %>% 
  count()
ANSWER count() works like summarise(n = n())



What is this code equivalent to?

diamonds %>% 
  count(cut)
ANSWER count(cut) works like group_by(cut) %>% summarise(n = n())



What is this code equivalent to?

diamonds %>% 
  count(cut, color)
ANSWER count(cut) works like group_by(cut, color) %>% summarise(n = n())



What is this code equivalent to?

diamonds %>% 
  add_count()
ANSWER add_count() works like mutate(n = n())

#### Difference between `==` and `%in%`

tldr - Use `%in%` instead of `==` when you want to filter for multiple values.  
  
Read on if you want to understand why.  (You don't have to)

The code `filter(Ethnicity == c("Mexican", "Filipino"))` is likely not doing what you expect. The '==' operator does an element-wise comparison, which means it compares the first element of 'Ethnicity' to the first element of the vector ("Mexican"), the second element of 'Ethnicity' to the second element of the vector ("Filipino"). The short vector is then recycled so now the third element of 'Ethnicity' is compared to the first element of the vector ("Mexican"), the fourth element of 'Ethnicity' to the second element of the vector ("Filipino"), and so on.

## `count()`

For simple counting `count()` is faster than `summarise(n = n())` or `mutate(n = n())`

<br><br>

### What is this code equivalent to?
```{r}
#| eval: false
#| code-fold: false
diamonds %>% 
  count()
```

<details><summary>ANSWER</summary>
`count()` works like `summarise(n = n())`
</details>

<br><br>

### What is this code equivalent to?
```{r}
#| eval: false
#| code-fold: false
diamonds %>% 
  count(cut)
```

<details><summary>ANSWER</summary>
`count(cut)` works like `group_by(cut) %>% summarise(n = n())`
</details>

<br><br>

### What is this code equivalent to?
```{r}
#| eval: false
#| code-fold: false
diamonds %>% 
  count(cut, color)
```

<details><summary>ANSWER</summary>
`count(cut)` works like `group_by(cut, color) %>% summarise(n = n())`
</details>

<br><br>

### What is this code equivalent to?
```{r}
#| eval: false
#| code-fold: false
diamonds %>% 
  add_count()
```

<details><summary>ANSWER</summary>
`add_count()` works like `mutate(n = n())`
</details>
